# LiCO 直播伴侣


## 一、项目简介

通过直播做自媒体成为了越来越多年轻人谋生的选择，但在缺乏指导以及对冷冰冰的数据无感的情况下，很多人比较迷茫不知道自己应该播什么样的内容，不知道如何应对观众的反馈。尽管MCN的出现可以部分解决这一问题，但仍有大量的主播不愿意签约MCN，他们或是不愿面对交付高额的分成，或是与MCN之间存在信任危机。为了解决这一痛点，我们提出 Live COmpanion （LiCO 直播伴侣），这是全球第一个直播Agent软件，通过捕捉直播间的多模态信息，直播可以分析出目前主播以及观众的心理活动，并在主播下播之后开到主播的心态，为主播充能，对未来的直播内容进行指导，以及一些使用服务（比如自动切片）。本项目基于开源项目 [Blivechat](https://github.com/xfgryujk/blivechat/) 以及 [OBS-Studio](https://github.com/obsproject/obs-studio) 进行开发，在此基础上增加了 AI Agent 去识别弹幕/音频/画面的信息，结合蒸馏的数据库进行共享上下文的联合多模态分析。


## 二、交付件简介

交付的代码分为客户端（见client文件夹）以及服务端（见server文件夹）。客户端以及OBS Studio运行在主播的电脑上，服务端则运行在AI服务器上，开启直播时，客户端会自动接受到推送，并静默完成AI分析。客户端推荐配置：Windows 10/11 64 位，6 核 CPU，16GB 内存，有独显更好，主要用于 OBS 编码/直播稳定性，上行 10 Mbps+，使用有线网络优先。服务端推荐配置：GPU: NVIDIA H800 PCIe 80GB，显存: 80GB 级别，CPU: 16 核或以上，内存: 64GB 或以上，磁盘: 100GB 以上可用空间，OS: Ubuntu 22.04 / Linux 5.15+，Driver: 580.82.07 或兼容版本，CUDA: 13.0 或兼容 CUDA 12/13 运行环境。


## 三、客户端运行教程（详见/client文件夹的README文件）

1. 启动 Python 后台程序，后台程序会开始弹幕串流，但是不会启动AI Agent进行分析。

python main.py --host 127.0.0.1 --port 12450 

2. 按照弹出的窗口提示输入房间号进入B站直播监听浏览器窗口，记住这个窗口。

3. 下载OBS-studio

4. 将插件obs-blivechat-bridge安装到OBS中。

    Windows：将 `obs-blivechat-bridge` 下编译出的 `dll` 和 `pdb` 文件复制到 OBS 安装目录下：

C:\Program Files\obs-studio\obs-plugins\64bit

    Mac：仓库里的 bridge 现在支持编译成 `.plugin` bundle。请用 macOS 的 CMake 和 OBS 开发包编译 `obs-blivechat-bridge`，然后把生成的 `obs-blivechat-bridge.plugin` 安装到 `~/Library/Application Support/obs-studio/plugins/`。

5. 打开OBS，点击开始直播，然后在 2 的窗口中就可以看到插件的效果。

## 四、服务端运行教程（详见/server文件夹的README文件）

推荐直接运行：

```bash
cd /root/Gemma
bash start_gemma_api.sh
```

脚本会：

- 设置 Hugging Face 国内镜像
- 设置 Ollama 并发和 KV cache 参数
- 设置模型名
- 设置 browser-use/search 参数
- 设置 CUDA 12 动态库路径
- 确保 Ollama 正在运行
- 后台启动 FastAPI/Uvicorn
- 将日志写入 `gemma_YYYYMMDD_HHMMSS.log`

默认监听：

```text
0.0.0.0:6006
```

# 其他事项说明

## 文件名中带有Qwen？

这是一个命名错误，因为之前使用了和Qwen相同的Open AI 格式的API接口并接入了Qwen，目前的客户端API没有太大的改变而服务端变为了Gemma，本项目客户端以及服务端代码的一切测试在 Gemma 4 31B 模型下进行。
