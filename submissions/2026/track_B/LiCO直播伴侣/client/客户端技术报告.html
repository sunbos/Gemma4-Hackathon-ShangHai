一、模型选型理由

模型/策略使用位置选择理由
gemma4-31b-mm联合心理/语音/画面分析与视觉场景31B 规模给足复杂推理和多模态融合能力，适合把截图、双路音频和弹幕统计合并成一个结构化 JSON。
gemma4-e4b-mmASR/轻量语音观察E4B 规格更轻，用于单独语音转写和声音分类，降低延迟和资源占用。
规则分析 fallback无模型或非结构化返回时保障 UI 不空白；当模型返回英文 reasoning 或非 JSON 时，只展示本地中文兜底结果。

选择 gemma4-31b-mm 的核心理由是联合分析任务不是单纯 OCR 或 ASR，而是需要同时理解截图、弹幕统计、麦克风音频、桌面音频和直播语境。31B 规模提供更强的跨模态归纳和结构化输出能力。对单独 ASR/语音观察，gemma4-e4b-mm 更轻，适合降低成本与延迟。

当前客户端策略是：业务 UI 只展示 delta.content 或可解析 JSON；delta.reasoning 只写后端 debug 日志，避免英文推理过程污染用户界面。

三、Agent 设计

实时 Room Agent
  • Room.vue 每个窗口聚合弹幕、礼物、SC、OBS 状态，调用 /api/qwen/joint-report，再分发到心理、语音、礼物、画面四个模块。
  • services/psych_agent.py 是后端常驻心理状态服务，可从弹幕事件生成窗口 metrics，并调用 Qwen 或规则 fallback。
  • agent_pipeline.py 是命令行离线/实验 Agent，连接 /api/chat 聚合窗口，调用 /api/qwen/chat 验证提示词和指标体系。
  • Agent 输出协议包含 stress、valence、fatigue、engagement、confidence、danmakuDistill、liveEnvironment、voiceReport、sceneReport。
输出分发
  • 心理模块：压力、积极度、疲劳、参与度、弹幕蒸馏。
  • 语音模块：转写、声音类型、活动、外显状态。
  • 礼物模块：礼物、舰长、SC、累计价值。
  • 画面模块：游戏中、唱歌中、观看视频中及证据。

四、OBS 插件设计

obs-blivechat-bridge
  • OBS 插件固定上报到 http://127.0.0.1:12450，避免跨机网络和认证复杂度。
  • 事件通道上报 streaming_started / streaming_stopped、录制路径、推流时长。
  • 画面通道上报最新截图或截图路径；后端再压缩为 JPEG data URL。
  • 音频通道采样 16kHz 单声道 WAV base64，支持 mixed、mic、desktop 三种 trackRole。
  • 后端 obs_bridge 保存最新状态，并通过 /api/obs-agent WebSocket 广播给前端。

设计取舍：OBS 插件只负责采集和上传，不做 AI 推理；后端统一保存最新状态并决定何时调用模型。这样可以把 OBS 端复杂度控制在 C++ 插件可维护范围内，同时让 Python/Vue 侧更容易迭代分析策略。

六、直播归档与复盘链路

写入侧
  • Room.vue 在四模块结果生成后调用 /api/live/archive/append。
  • 分类文件包括 mental.txt、voice.txt、picture.txt、gift.txt、live_info.txt。
  • 推流开始时通过 /api/live/archive/init 创建直播 session。
读取侧
  • LiveReviewDetail.vue 读取 session、clip timeline、scene timeline。
  • services/live_archive.py 解析归档文本，生成场景时间轴和高光切片建议。
  • 可用 ffmpeg/ffprobe 抽帧生成封面、读取录播时长。

八、当前架构的关键质量措施

上传链路诊断

api/qwen_multimodal.py 记录 body 大小、首 SSE 字节、首文本 token、content/reasoning chunk 数量，并在上传错误中计算近似上传网速。

非 JSON 容错

Room.vue 会尝试从响应文本中提取 JSON；解析失败时只显示中文规则兜底，不展示 reasoning 原文。

插件隔离

插件由子进程启动，并通过 Bearer token + WebSocket 接入；插件异常不会直接拖垮主 Tornado 服务。

九、建议

  1. 把 /api/qwen/joint-report 的输出 schema 固化为版本化协议，例如 jointReport.v1，并在前后端共享字段约束。
  2. 将 OBS 上传链路从 AutoDL :8443 网关逐步迁移到更稳定的本地隧道或对象存储中转，减少 0.5MB 级请求的抖动。
  3. 将 Room.vue 中 Agent 编排逻辑拆到独立 composable/service，降低单文件复杂度。
  4. 为 content/reasoning、非 JSON、空响应、413、ReadError 等路径补充自动化回归测试。
