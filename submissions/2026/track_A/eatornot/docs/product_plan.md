# EatOrNot 产品计划

## 项目定位

EatOrNot 是一个基于多 Agent 协作的智能饮食决策助手。它不是简单的麦当劳 MCP 包装器，而是一个完整的营养、预算、情绪管理决策系统。

## 核心理念

**"不替你做决定，帮你做更好的决定"**

- 不简单说"不许吃"
- 让多个 Agent 从不同角度分析
- 提供多个方案让用户自主选择
- 透明展示每个方案的利弊

## 用户场景

**典型用户**：忙碌的上班族/学生

- 正在减脂
- 预算有限
- 偶尔想吃麦当劳
- 需要快速决策

**典型场景**：
> "我想吃麦当劳，我在减肥但是好累，预算不多"

## Agent 角色

| Agent | 职责 |
|-------|------|
| Profile Agent | 加载用户档案，推断偏好 |
| Weight Loss Agent | 计算 BMI/BMR/TDEE，设定热量目标 |
| Nutrition Agent | 评估食物营养质量 |
| Budget Agent | 检查预算，推荐性价比方案 |
| Craving Agent | 检测情绪化进食，允许适度放纵 |
| Time Context Agent | 根据时间/压力推荐 |
| Menu Agent | 调用 MCP 或 Mock 数据 |
| Safety Agent | 过敏检查，安全警告 |
| Future Simulation Agent | 模拟后续影响 |

## 输出形式

每个推荐包含：
1. **自律减脂餐** — 低热量高蛋白
2. **省钱包饱餐** — 最高性价比
3. **放纵一下餐** — 满足口腹之欲

每个方案附带：
- 价格 / 热量 / 营养素
- 预算影响
- 热量影响
- 放纵额度影响
- 优缺点
- Agent 投票结果

## 安全约束

- 不自动下单，必须用户确认
- 不提供医疗诊断
- 极端饮食目标会被警告
- 过敏信息会被优先检查
