# EatOrNot Demo 脚本

## 场景设定

**用户**：小明，24岁上班族
- 身高 172cm，体重 72kg
- 目标：减脂
- 日预算：¥45
- 每周放纵额度：2次
- 现在：晚上 7 点，加班很累，想吃麦当劳

## Demo 流程

### Step 1: 启动系统

```bash
# Terminal 1: 启动后端
cd apps/api
pip install -r requirements.txt
uvicorn main:app --reload --port 8000

# Terminal 2: 启动前端
cd apps/web
npm install
npm run dev
```

### Step 2: 打开浏览器

访问 http://localhost:5173

### Step 3: 查看用户档案

左侧面板自动加载 Demo 用户信息：
- 身高 172cm / 体重 72kg
- 目标：减脂
- 日预算：¥45

### Step 4: 输入需求

在输入框中输入：
> "我想吃麦当劳，我在减肥但是好累，预算不多"

点击"🔍 开始分析"

### Step 5: 查看 Agent 辩论

右侧面板显示 8 个 Agent 的分析结果：

| Agent | 观点 |
|-------|------|
| Profile Agent | ✅ 用户目标：减脂，BMI 正常 |
| Weight Loss Agent | ✅ 推荐低热量选项，避免含糖饮料 |
| Nutrition Agent | ✅ 推荐板烧鸡腿堡 + 玉米杯 |
| Budget Agent | ✅ 平均消费在预算 60% 以内 |
| Craving Agent | ✅ 检测到疲劳，允许适度放纵 |
| Time Context Agent | ✅ 快餐适合当前时间紧迫场景 |
| Safety Agent | ✅ 无过敏冲突 |
| Future Simulation Agent | ✅ 剩余热量充足，明天可平衡 |

### Step 6: 查看推荐方案

中部显示 3 个推荐卡片：

**💪 自律减脂餐**
- 板烧鸡腿堡 + 玉米杯 + 零度可乐
- ¥33 / 460 kcal
- 蛋白质 27.5g / 钠 880mg
- 预算 73% / 热量 23%

**💰 省钱包饱餐**
- 薯条(小) + 玉米杯 + 美式咖啡
- ¥25.5 / 315 kcal
- 蛋白质 5.8g / 钠 215mg
- 预算 57% / 热量 16%

**🍔 放纵一下餐**
- 麦辣鸡腿堡 + 巨无霸 + 麦旋风
- ¥53 / 1350 kcal
- 蛋白质 55g / 钠 2190mg
- 预算 118% / 热量 68%

### Step 7: 选择方案

点击"选择此方案"按钮，选中"自律减脂餐"

### Step 8: 确认下单

点击"🛒 确认下单"，弹出确认对话框：
- 显示选中食物
- 显示价格和热量
- 显示预算影响
- 显示放纵额度影响
- 要求用户点击"确认下单"

### Step 9: 查看结果

显示订单确认信息：
> 订单 MOCK-12345 已模拟创建！This is a simulated order (no real MCP token).

## 关键展示点

1. **多 Agent 协作**：不是单一决策，而是 8 个 Agent 各自分析
2. **透明辩论**：用户可以看到每个 Agent 的观点和评分
3. **多方案选择**：不强制推荐，提供 3 个不同侧重的方案
4. **安全约束**：必须用户确认才能下单
5. **Mock 兜底**：没有 MCP token 也能完整演示

## 演讲要点

> "EatOrNot 不是简单的麦当劳 MCP 包装器。它是一个多 Agent 决策系统，从营养、预算、情绪、时间、安全等多个维度帮你做出更好的饮食选择。"

> "我们不替你做决定，我们帮你做更好的决定。"

> "每个 Agent 都有独立的观点，系统会透明地展示辩论过程，让你了解每个建议背后的理由。"
