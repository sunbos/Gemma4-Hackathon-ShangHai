### [0:40 - 1:20] 架构速览

**画面**：架构图动画

**旁白**：
> EatOrNot 的核心不是菜单推荐，而是一个多 Agent 决策系统。
>
> 8 个专业 Agent 并行分析：
> 档案 Agent 了解你的身体数据，
> 营养 Agent 评估每一餐的营养素，
> 预算 Agent 守住你的钱包，
> 食欲 Agent 理解你的情绪，
> 安全 Agent 检查过敏风险。
>
> 它们不是各自为政——
> 而是通过 **4 阶段圆桌辩论**：陈述立场、发现冲突、形成妥协、最终投票。
>
> Orchestrator 利用 **Gemma 4 的原生 Function Calling** 能力，动态选择最相关的 Agent。
> 每个 Agent 都有独立的知识库支撑，基于《中国居民膳食指南》等权威数据。
>
> 关键是——这一切都在你的终端里完成。

**画面**：切换到终端，展示架构简图

```
用户 → SupervisorAgent → OrchestratorAgent (Gemma 4)
                              ├── 档案Agent  ├── 减脂Agent
                              ├── 营养Agent  ├── 预算Agent
                              ├── 食欲Agent  ├── 安全Agent
                              └── ...并行分析...
                         → DebateEngine (4阶段辩论)
                         → 3 个方案: 自律 / 省钱 / 犒劳
```

### [1:20 - 2:30] 演示 1：主动提醒 + AI 推荐

**画面**：终端 Claude Code 会话，突然弹出提醒

**旁白**：
> EatOrNot 会通过 Cron 定时任务，在早餐、午餐、晚餐时间主动提醒你。
> 不需要你记得吃饭——它记得。

**终端操作**：
```
🌅 早餐时间到了！执行 meal-order skill...

🍽️ 智囊团推荐方案

方案 1: 💪 自律减脂餐
  猪柳麦满分 + 无糖美式咖啡
  💰 ¥22 | 🔥 380kcal | 蛋白质 24g
  ✅ 低热量，高蛋白，有助减脂目标

方案 2: 💰 省钱包饱餐
  吉士蛋麦满分 + 美式咖啡
  💰 ¥18 | 🔥 420kcal
  ✅ 最实惠，依然管饱

方案 3: 🍔 犒劳一下餐
  双层吉士汉堡套餐
  💰 ¥35 | 🔥 680kcal
  ✅ 偶尔犒劳，消耗1次放纵额度
```

**旁白**：
> 三套方案，三种优先级。你选哪个？
> 每个方案背后都是多个 Agent 的分析结果——
> 不只是"推荐热销"，而是基于你的个人档案做的权衡。

**画面**：选择方案 1

```
> 我选方案1

✅ 已选择：💪 自律减脂餐
要帮你下单吗？(y/n)
```

## 🎬 录制建议

### 环境准备
- 终端：深色主题（推荐 One Dark 或 Dracula），字体清晰
- 浏览器：Chrome 全屏，关闭无关标签页
- 提前准备好 demo 数据（建档完成，有历史用餐记录）

### 演示前预跑
1. 确保后端运行在 `localhost:8000`
2. 确保前端运行在 `localhost:5173`
3. 先跑一次完整推荐流程，确保 LLM 响应正常
4. 终端 Claude Code 已安装 meal-order skill

### 录制工具
- macOS: QuickTime Screen Recording 或 OBS
- Windows: OBS Studio 或 Windows Game Bar (Win+G)
- 分辨率: 1920×1080，30fps 足够

### 节奏控制
- 终端操作适当放慢，让文字有时间展示
- API 响应等待时可以切入旁白
- 关键决策点（选方案、确认下单）加短暂停顿
