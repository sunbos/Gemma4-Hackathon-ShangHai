# Emoti-Gotchi 产品与技术架构评审报告

版本：提交前产品评审版  
评审角色：CEO / 产品负责人 / 技术架构师  
线上演示：https://emoti-gotchi.vercel.app

## 1. 执行摘要

Emoti-Gotchi 是一套面向 6-8 岁儿童家庭的隐私优先端侧情绪支持系统。

产品希望解决的核心问题是：低龄儿童经常无法准确表达害怕、焦虑、委屈等感受，家长也难以从一句“我没事”中识别隐藏情绪。Emoti-Gotchi 通过儿童端的陪伴式反馈和家长端的情绪天气摘要，帮助家庭更早发现异常信号，并以低压力方式开始沟通。

本产品不是心理诊断工具，不替代家长、教师或专业心理服务。其定位是：

> 在保护儿童隐私的前提下，提供即时、温和、可解释的情绪支持与家长行动建议。

当前版本已经完成可在线演示的产品闭环，包括孩子端反馈、家长端摘要、安全熔断、Gemma 4 云端复盘、端侧 E2B 推理模拟和结构化技术证据展示。

## 2. 产品价值主张

### 对儿童

- 通过表情、灯光、声音和简短语言提供即时陪伴。
- 不要求儿童必须准确描述复杂情绪。
- 在产品目标架构中，原始语音和声学特征优先留在端侧。
- 高风险场景不会让模型自由处理，而是进入确定性的安全流程。

### 对家长

- 将复杂信号转化为简单的“情绪天气”：不是给孩子贴标签，而是告诉家长此刻适合分享、安静陪伴、降低刺激还是进入明确安全流程。
- 提供“现在应该做什么”和“应该避免什么”。
- 不直接展示原始语音和完整对话，减少儿童被监控的感觉。
- 高风险时先弹出家长安全提醒与即时行动步骤；连接经过审核的儿童心理专业支持必须经过家长明确同意。

### 对社会价值

- 帮助家庭更早注意儿童隐藏 distress。
- 降低儿童表达情绪的语言门槛。
- 推动隐私优先、非诊断式的儿童 AI 产品设计。

## 3. 核心产品流程

1. 儿童表达一句话，设备在本地获得语言和模拟声学特征。
2. 实时端侧路径判断当前情绪，并输出受约束的 JSON 动作。
3. 儿童端立即通过表情、灯光、声音和温和语言进行反馈。
4. 确定性安全审计检查高风险语言和极端 distress 信号。
5. 家长端收到简化的情绪天气、行动建议和避免事项，避免直接暴露儿童原话或复杂分数。
6. 云端 Gemma 4 在后台总结多日情绪天气，指出哪些时间和家庭环境更需要安静、柔和或可预测的陪伴。
7. 高风险信号退出天气表达，直接通知家长；家长授权后才进入儿童心理专业支持转介流程。

## 4. Gemma 4 的必要性

Gemma 4 在本产品中不是通用聊天机器人，而是结构化情绪推理层。

普通关键词规则可以识别明显表达，例如“我很害怕”。但它难以处理语言与状态之间的矛盾，例如：

> “我没事，只是房间太黑了，我不敢一个人睡。”

Gemma 4 的价值在于：

- 理解一句话中的隐含含义和矛盾。
- 综合语言、声学压力、呼吸和哭泣概率等信号。
- 生成固定结构的儿童端动作和家长端建议。
- 为家长提供更可解释、非诊断式的复盘。

## 5. 当前技术架构

```text
儿童语言 + 本地声学特征
        |
        +--> 实时端侧路径
        |     Gemma 4 E2B Edge Sim
        |     结构化 JSON 动作
        |     确定性安全审计
        |     表情 / 灯光 / 声音 / 儿童回应
        |
        +--> 后台云端路径
              Gemma 4 Cloud Review
              家长复盘 / 趋势理解 / 行动建议
```

## 6. 当前产品完成度判断

当前版本属于：

> 可参加比赛路演的高保真在线产品原型，已具备真实云端 Gemma 调用和完整产品叙事，但尚未达到真实儿童试点或硬件量产状态。

已完成的主要成果：

- 产品定位和社会问题定义清晰。
- 儿童端、家长端和安全流程已经形成闭环。
- Gemma 4 的作用可以通过 Demo 解释。
- 云端模型真实接入。
- 端侧架构、隐私边界和未来硬件路线已经明确。
- 在线 Demo、README、技术报告和评分对齐材料齐全。

## 7. CEO 需要重点检查的问题

### 7.1 产品定位是否足够聚焦

建议坚持“儿童隐私优先的端侧情绪支持系统”，避免重新退回“AI 情绪玩具”的表达。

需要决策：

- 第一目标用户是否锁定 6-8 岁儿童家庭。
- 第一落地场景是家庭睡前陪伴、日常情绪表达，还是学校场景。
- 产品首先服务家长，还是首先服务儿童体验。

建议：比赛版本聚焦家庭睡前和日常情绪表达，避免扩展到学校、医疗和诊断。

### 7.2 用户是否愿意信任产品

主要风险：

- 家长可能把产品误解为监控工具。
- 儿童可能因为被分析而降低表达意愿。
- 家长可能过度相信模型建议。

建议：

- 强调原始语音不上传的目标架构。
- 家长只查看摘要，不查看完整私密对话。
- 所有页面持续强调“支持而非诊断”。

### 7.3 商业与社会价值是否能够同时成立

当前社会价值表达强，但商业模式尚未验证。

后续可验证方向：

- 家庭硬件设备加基础订阅。
- 与儿童成长、家庭教育品牌合作。
- 提供本地化的家庭情绪记录和家长指导内容。

不建议比赛阶段讨论医疗收费、诊断服务或专业治疗替代。

## 8. 架构师需要重点检查的问题

### 8.1 端侧可行性尚未完成真实验证

当前 E2B Edge 路径为模拟，不等同于模型已运行在 Raspberry Pi。

必须补充的真实证据：

- 目标模型和量化方式。
- 首 Token 延迟与完整输出延迟。
- 峰值内存占用。
- 发热、功耗和连续运行稳定性。
- 端侧不可用时的降级策略。

### 8.2 多模态输入仍是概念验证

当前声学压力、呼吸和哭泣概率主要通过 UI 模拟。

后续需要定义：

- 声学特征由什么模型或算法生成。
- 特征是否可以完全在本地提取。
- 如何减少环境噪声、电视声音和其他家庭成员造成的误判。
- 是否保存历史特征，以及保存多久。

### 8.3 安全策略需要专业审查

当前确定性安全审计优于完全依赖模型，但规则本身仍需儿童安全专家验证。

关键风险：

- 漏报高风险状态。
- 误报导致家长恐慌。
- 错误建议造成二次压力。
- 高风险场景缺少地区化支持资源。

### 8.4 云端模型调用需要生产级保护

当前已经实现服务端 Key 隔离，但真实产品仍需要：

- 请求速率限制。
- 成本与配额监控。
- 日志脱敏。
- 超时、重试和熔断机制。
- 内容安全与提示注入防护。
- 云端不可用时的明确降级体验。

### 8.5 数据与隐私设计需要形成正式规范

比赛版本表达了“原始语音留在本地”，但尚未形成完整的数据生命周期设计。

需要明确：

- 收集哪些数据。
- 数据存储位置和保存周期。
- 家长如何查看、导出和删除数据。
- 儿童隐私权和家长知情同意如何平衡。
- 如何满足未来目标地区的儿童隐私法规。

## 9. 最高优先级风险清单

| 风险 | 当前状态 | 影响 | 建议 |
|---|---|---|---|
| 端侧能力仍为模拟 | 未解决 | 技术可信度 | 尽快完成一轮真实硬件 benchmark |
| 儿童真实用户验证不足 | 未解决 | 社会价值与产品有效性 | 先进行家长和专家访谈，不直接开展儿童测试 |
| 云端响应较慢 | 已识别 | 即时体验 | 坚持 Edge 即时响应、Cloud 后台复盘 |
| 安全规则未经过专业审查 | 未解决 | 儿童安全 | 引入儿童心理或安全专家评审 |
| 产品可能被理解为监控 | 已识别 | 用户信任 | 强化本地处理、摘要展示和删除权 |
| 模型建议可能被误解为诊断 | 部分控制 | 法律与伦理 | 页面和报告持续明确非诊断定位 |

## 10. 提交前产品优化建议

### P0：必须完成

- 确保线上 Demo 三个关键场景稳定。
- 明确显示 Edge Sim 与真实 Cloud Review 的区别。
- Cloud Review 等待期间展示清晰状态，避免用户认为页面卡死。
- 检查所有文案，避免出现诊断或已经真实运行在树莓派上的错误表述。
- 完成 5 分钟以内演示视频。

### P1：强烈建议完成

- 强化 Rules vs Gemma 对比，使评委看见 Gemma 的必要性。
- 增加端侧可行性面板，明确目标设备、内存预算和延迟目标。
- 优化首屏定位，让评委在 10 秒内理解产品价值。
- 检查移动端布局和演示稳定性。

### P2：比赛后推进

- 完成 Raspberry Pi 或移动端真实 benchmark。
- 进行家长访谈和儿童安全专家评审。
- 建立正式数据隐私规范。
- 接入真实麦克风和本地声学特征提取。
- 设计实体硬件原型。

## 11. CEO 与架构师决策清单

提交前需要确认：

- [ ] 是否坚持“家庭场景、6-8 岁儿童”的聚焦定位。
- [ ] 是否明确产品不是诊断、治疗或监控工具。
- [ ] 是否接受当前 E2B 为透明模拟，而非真实硬件部署。
- [ ] 是否将 Cloud Review 定位为后台家长复盘，而非实时儿童回应。
- [ ] 是否所有高风险流程都由确定性安全规则控制。
- [ ] 是否允许在缺少专业审查前进行真实儿童试点。
- [ ] 是否准备好向评委解释隐私边界、误判风险和下一阶段计划。

## 12. 最终评审结论

Emoti-Gotchi 当前最强的竞争力不是功能数量，而是清晰的社会问题、隐私优先架构、端云职责拆分和温和的家庭支持体验。

当前最大短板是真实端侧验证和真实用户证据不足。因此，比赛提交时必须保持技术表达诚实：

- 云端 Gemma 4 已真实接入。
- E2B 端侧路径当前为可观测模拟。
- 产品流程、安全设计和端侧技术路线已经得到验证。
- 下一步是完成真实硬件 benchmark 与专业用户验证。

在保持以上边界的前提下，当前版本已经具备完整、可信、可演示的参赛产品形态。
